Sebagian komponen dari **ommiputera.com** dibangun menggunakan [**Shadcn/UI**](https://ui.shadcn.com/).

### Alasan utama

Shadcn/UI memberikan fleksibilitas dalam menyesuaikan komponen-komponennya agar sesuai dengan _design system_.

Jadi, apa artinya? Mari kita bahas..

### Kontrol penuh

Dengan Shadcn/UI, saya memiliki kontrol penuh atas kode setiap komponen yang saya gunakan. Ini berarti saya dapat:

- Kustomisasi Mendalam: Saya dapat menyesuaikan komponen secara detail untuk mencocokkan _design system_ apa pun.
- Pengembangan Tanpa Batasan: Fitur-fitur baru dapat ditambahkan tanpa terikat pada batasan bawaan _library_.
- Optimasi UI: Menjamin performa tinggi, aksesibilitas, dan kemudahan _maintainance_ dalam jangka panjang.

Berbeda dengan _framework_ atau _library_ UI seperti [**Mantine**](https://mantine.dev/) dan [**Bootstrap**](https://getbootstrap.com/) yang mengharuskan saya untuk menyesuaikan desain dengan aturan mereka. Shadcn/UI memberikan kebebasan penuh untuk menciptakan sesuatu yang benar-benar sesuai kebutuhan project. Saya tidak lagi merasa terbatas atau kesulitan saat memodifikasi komponen maupun styling.

### Modularitas

> Hanya menginstall komponen yang dibutuhkan pada project

Salah satu keunggulan terbesar Shadcn/UI adalah pendekatan modularnya. Tidak seperti _framework_ atau _library_ UI lain yang yang mengharuskan instalasi seluruh paket komponen, Shadcn/UI memungkinkan saya hanya menginstal komponen yang benar-benar relevan.

Pendekatan ini memberikan manfaat yang signifkan pada project seperti:

- Hemat Ruang Penyimpanan: Tidak perlu menginstal semua komponen yang tidak dibutuhkan.
- Efisiensi Pengembangan: Fokus hanya pada fitur yang penting untuk project.

Contoh: Untuk menambahkan komponen Input:

```sh
npx shadcn@latest add input
```

Dengan hanya menjalankan CLI di atas, saya mendapatkan komponen Input yang aksesibel, modern, dan memiliki desain minimalis yang siap digunakan.

### Implementasi

Setiap proyek memiliki kebutuhan yang unik, termasuk dalam penggunaan komponen. Dengan Shadcn/UI, saya memiliki fleksibilitas penuh untuk mengkustomisasi komponen Input sesuai dengan kebutuhan proyek, baik dari segi desain, fungsionalitas, maupun aksesibilitas. Berikut adalah langkah-langkah dan contoh implementasinya.

```tsx
import * as React from "react";

import { cn } from "@/lib/utils";

const Input = React.forwardRef<HTMLInputElement, React.ComponentProps<"input">>(
  ({ className, type, ...props }, ref) => {
    return (
      <input
        type={type}
        className={cn(
          "flex h-9 w-full rounded-md border border-input bg-transparent px-3 py-1 text-base shadow-sm transition-colors file:border-0 file:bg-transparent file:text-sm file:font-medium file:text-foreground placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring disabled:cursor-not-allowed disabled:opacity-50 md:text-sm",
          className,
        )}
        ref={ref}
        {...props}
      />
    );
  },
);
Input.displayName = "Input";

export { Input };
```

Bagaiman kalo saya ingin menambahkan ikon mata untuk melihat password?

```tsx
import * as React from "react"

import { cn } from "@/lib/utils"

const Input = React.forwardRef<HTMLInputElement, React.ComponentProps<"input">>(
  ({ className, type, ...props }, ref) => {
    let [showPassword, setShowPassword] = React.useState(false);
    let togglePasswordVisibility = () => {
      setShowPassword(!showPassword);
    };

    let inputType = type === "password" && showPassword ? "text" : type;
    return (
      <div className="relative">
        <input
          type={inputType}
          className={cn(
            "flex h-9 w-full rounded-md border border-input bg-transparent px-3 py-1 text-base shadow-sm transition-colors file:border-0 file:bg-transparent file:text-sm file:font-medium file:text-foreground placeholder:text-muted-foreground focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-ring disabled:cursor-not-allowed disabled:opacity-50 md:text-sm",
            className
          )}
          ref={ref}
          {...props}
        />
        {type === "password" && (
          <div
            role="button"
            className="focus-visible:outline-primary-500 absolute right-2 top-1/2 flex h-full w-11 -translate-y-1/2 items-center justify-center rounded-md"
            onClick={togglePasswordVisibility}
          >
            {showPassword ? (
              // icons.visibility
            ) : (
              // icons.visibility_off
            )}
          </div>
        )}
      </div>
    )
  }
)
Input.displayName = "Input"

export { Input }
```

### Kesimpulan

Dengan fleksibilitas tinggi, kontrol penuh, dan pendekatan modular, Shadcn/UI telah menjadi salah satu alat favorit saya dalam membangun ommiputera.com. Hasilnya adalah sebuah aplikasi yang tidak hanya terlihat menarik, tetapi juga berjalan dengan baik, mudah diakses, dan gampang untuk dirawat di masa mendatang.

### **Proses Iterasi Development**

Pembuatan website dilakukan melalui pendekatan iteratif untuk memastikan hasil akhir sesuai dengan kebutuhan pengguna. Proses ini terdiri dari:

1. **Interview & Riset** – Memahami kebutuhan klien melalui diskusi awal dan riset untuk menemukan solusi terbaik.
2. **Perencanaan & Wireframing** – Menyusun struktur dan rancangan awal website agar sesuai dengan kebutuhan bisnis atau personal klien.
3. **Development Bertahap** – Pengembangan dimulai dari fitur utama terlebih dahulu, lalu disempurnakan melalui iterasi berulang.
4. **Uji Coba & Revisi** – Website diuji untuk memastikan fungsionalitasnya berjalan optimal sebelum di-launching.
5. **Launch & Serah Terima** – Setelah website siap, saya akan membantu proses deployment dan serah terima kepada klien.

```plaintext
Interview & Riset → Perencanaan & Wireframing → Development Bertahap → Uji Coba & Revisi → Launch & Serah Terima
```

### **Teknologi yang Digunakan**

Saya menggunakan stack modern untuk memastikan website cepat, responsif, dan mudah dikelola:

- **Next.js** – Framework React yang cepat dan SEO-friendly
- **React** – Library JavaScript populer untuk pengalaman pengguna yang interaktif
- **Tailwind CSS** – CSS framework yang efisien dan fleksibel untuk tampilan yang modern
- **Shadcn UI** – Komponen UI yang simpel dan elegan untuk pengalaman pengguna yang lebih baik

---

### **Flow Penyerahan Project**

Setelah website selesai dibuat, berikut adalah proses penyerahannya:

1. **Pembuatan Akun Hosting** – Klien diminta untuk membuat akun di [Vercel](https://vercel.com/) sebagai layanan serverless hosting agar website dapat diakses dengan optimal.
2. **Pembuatan Akun GitHub** – Klien juga diminta untuk membuat akun di [GitHub](https://github.com/) sebagai repositori kode sumber (source code) untuk pengelolaan lebih lanjut.
3. **Deploy ke Vercel** – Kode akan diunggah ke GitHub, lalu diintegrasikan dengan Vercel untuk deployment otomatis.
4. **Konfigurasi Domain** – Jika klien memiliki domain sendiri, saya akan membantu menghubungkannya ke Vercel.
5. **Serah Terima & Panduan** – Setelah deployment berhasil, saya akan memberikan panduan dasar untuk pengelolaan website, seperti update konten dan pengelolaan hosting.

📌 **Catatan:** Setelah serah terima, saya tidak menyimpan atau mengelola akun GitHub maupun Vercel klien.

shadcn/ui adalah kumpulan reusable components dengan pendekatan yang, menurut saya, sangat disukai oleh banyak developer yaitu "copy and paste". Namun, lebih dari itu, pendekatan ini juga bisa menjadi game changer bagi developer dalam membangun components library di project web applications.

Jadi, apa artinya? Mari kita bahas..

![I stole your code!](/images/meme/stole-code.jpeg)

### Kontrol penuh

Bagi developer yang familiar atau bekerja dengan React ecosystem, pasti pernah merasakan sulitnya membangun sebuah components library from scratch. "UI Component Library" seperti Material-UI, Mantine, dan Chakra UI menjadi solusi praktis untuk membangun components library. Namun, menyesuaikan functionality, UI, dan UX dengan design system bisa sangat melelahkan, hingga tak jarang components yang telah dibangun tidak sesuai dengan ekspektasi designer ðŸ˜­.

Dengan shadcn/ui, saya memiliki kontrol penuh terhadap kode pada setiap component, memutuskan bagaimana components pada project dibangun dan distyling. Dan tentu, saya memiliki kebebasan tanpa batas untuk mengembangkan setiap component sesuai dengan kebutuhan spesifik project.

### Modularitas

Sebagian besar components dalam shadcn/ui dibangun menggunakan headless component library seperti Radix Primitives, yang menawarkan dua fitur utama: accessibility dan unstyled components.

Setiap component yang menggunakan Radix Primitives sudah mematuhi standar Web Content Accessibility Guidelines (WCAG). Ini memastikan bahwa component tidak hanya fungsional, tetapi juga inklusif dan dapat diakses oleh end user, termasuk mereka yang menggunakan teknologi bantu seperti screen reader.

Pendekatan modular ini menjadikan shadcn/ui tidak hanya cepat diimplementasikan, tetapi juga sangat adaptif terhadap kebutuhan aplikasi tanpa perlu khawatir tentang kompleksitas tambahan yang sering muncul saat mencoba memenuhi standar aksesibilitas secara manual.

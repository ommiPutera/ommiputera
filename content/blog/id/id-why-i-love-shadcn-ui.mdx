[shadcn/ui](https://ui.shadcn.com/) merupakan kumpulan reusable components dengan pendekatan yang, menurut saya, sangat disukai oleh banyak developer yaitu "copy and paste." Namun, lebih dari itu, pendekatan ini juga menjadi game changer bagi developer dalam membangun sebuah UI (User Interface) components dalam pengembangan web applications.

Jadi, apa artinya? Mari kita bahas..

![I stole your code!](/images/meme/stole-code.jpeg)

## Kontrol Penuh

Bagi developer yang familiar atau bekerja dengan React ecosystem, pasti pernah merasakan sulitnya membangun sebuah UI components from scratch. "UI Component Libraries" seperti Material-UI, Mantine, dan Chakra UI menjadi solusi praktis untuk membangun components. Namun untuk menyesuaikan functionality, UI, dan UX dengan design system bisa sangat melelahkan, hingga tak jarang UI yang telah dibangun tidak sesuai dengan ekspektasi designer 😭.

Dengan shadcn/ui, saya memiliki kontrol penuh terhadap kode pada setiap component, memutuskan bagaimana UI components pada project dibangun dan distyling. Dan tentu, saya memiliki kebebasan tanpa batas untuk mengembangkan setiap component sesuai dengan kebutuhan spesifik project.

## Accessibility

Sebagian besar components dalam shadcn/ui dibangun menggunakan headless component library seperti [Radix Primitives](https://www.radix-ui.com/primitives), yang menawarkan dua fitur utama yaitu accessibility dan unstyled components.

Setiap component yang menggunakan Radix Primitives sudah mematuhi standar [Web Content Accessibility Guidelines](https://www.w3.org/TR/WCAG21/) (WCAG). Ini memastikan bahwa component tidak hanya fungsional, tetapi juga inklusif dan dapat diakses oleh end user, termasuk mereka yang menggunakan teknologi bantu seperti screen reader.

## Copy & Paste

Inilah daya tarik utama shadcn/ui: simplicity. Dengan pendekatan "copy and paste" atau menggunakan CLI, developer dapat langsung mengambil kode setiap component, memodifikasinya sesuai dengan design system, atau menggunakan default styling dari shadcn/ui, seperti gambar berikut:

![installation](/images/shadcn-installation.png)

Booom! "You have the component!"

Hal ini memungkinkan kita untuk menghindari constraint yang biasanya dimiliki oleh UI libraries.

Tidak perlu lagi menghabiskan waktu mempelajari konfigurasi spesifik library atau theming system yang terkadang terlalu kompleks, terutama jika Anda menggunakan lebih dari satu UI library pada project (shadcn is truly a lifesaver in situations like this 😂).

## TailwindCSS

shadcn/ui secara default menggunakan [TailwindCSS](https://tailwindcss.com/) sebagai styling framework. Bagi developer seperti saya, yang kurang menyukai menulis CSS secara manual, TailwindCSS adalah solusi ideal.

Keunggulan lainnya adalah kemudahan debugging, yang membuat proses pengembangan jadi lebih efisien. Kombinasi TailwindCSS dan shadcn/ui memberikan kekuatan besar dalam membangun UI.

## Community

Adopsi shadcn/ui dalam open-source community berkembang pesat, termasuk dukungan dari platform seperti [Vercel](https://vercel.com/). Ini membuat shadcn/ui semakin relevan dan terus bertumbuh 👍.

## Kesimpulan

Baru-baru ini, shadcn/ui memenangkan penghargaan [Most Popular Projects Overal](https://x.com/shadcn/status/1879560380336521537), mengukuhkan posisinya sebagai salah satu project paling berpengaruh dalam pengembangan aplikasi web.

shadcn/ui bukan hanya tentang kemudahan, tetapi juga tentang kebebasan, aksesibilitas, dan kekuatan komunitas. Jika Anda belum mencobanya, mungkin ini saat yang tepat untuk memasukkannya ke dalam toolkit pengembangan Anda.

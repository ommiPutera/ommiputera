[shadcn/ui](https://ui.shadcn.com/) merupakan kumpulan reusable components dengan pendekatan yang, menurutku, sangat disukai oleh banyak developer yaitu "copy and paste". Namun, lebih dari itu, pendekatan ini juga menjadi game changer bagi developer dalam membangun sebuah UI (User Interface) components dalam pengembangan web applications.

Jadi, apa artinya? Mari kita bahas..

![I stole your code!](/images/meme/stole-code.jpeg)

## Kontrol penuh

Bagi developer yang familiar atau bekerja dengan React ecosystem, pasti pernah merasakan sulitnya membangun sebuah UI components from scratch. "UI Component Library" seperti Material-UI, Mantine, dan Chakra UI menjadi solusi praktis untuk membangun components. Namun untuk menyesuaikan functionality, UI, dan UX dengan design system bisa sangat melelahkan, hingga tak jarang UI yang telah dibangun tidak sesuai dengan ekspektasi designer ğŸ˜­.

Dengan shadcn/ui, aku memiliki kontrol penuh terhadap kode pada setiap component, memutuskan bagaimana UI components pada project dibangun dan distyling. Dan tentu, aku memiliki kebebasan tanpa batas untuk mengembangkan setiap component sesuai dengan kebutuhan spesifik project.

## Accessibility

Sebagian besar components dalam shadcn/ui dibangun menggunakan headless component library seperti [Radix Primitives](https://www.radix-ui.com/primitives), yang menawarkan dua fitur utama yaitu accessibility dan unstyled components.

Setiap component yang menggunakan Radix Primitives sudah mematuhi standar [Web Content Accessibility Guidelines](https://www.w3.org/TR/WCAG21/) (WCAG). Ini memastikan bahwa component tidak hanya fungsional, tetapi juga inklusif dan dapat diakses oleh end user, termasuk mereka yang menggunakan teknologi bantu seperti screen reader.

## Copy & Paste

Inilah daya tarik utama shadcn/ui: simplicity. Dengan pendekatan "copy and paste" atau menggunakan CLI, developer dapat langsung mengambil kode setiap component, memodifikasinya sesuai dengan design system, atau menggunakan default styling dari shadcn/ui, seperti gambar berikut:

![installation](/images/shadcn-installation.png)

Booom! "You have the components!"

Hal ini memungkinkan kita untuk menghindari constraint yang biasanya dimiliki oleh UI libraries.

Tidak perlu lagi menghabiskan waktu mempelajari konfigurasi spesifik library atau theming system yang terkadang terlalu kompleks, terutama jika kamu menggunakan lebih dari satu UI library pada project (shadcn is truly a lifesaver in situations like this ğŸ˜‚).

## TailwindCSS

shadcn/ui by default menggunakan [TailwindCSS](https://tailwindcss.com/) sebagai styling framework, dan menurutku, ini adalah salah satu "Real Deal" yang membuat aku menyukai shadcn/ui, karena aku termasuk developer yang kurang menyukai menulis CSS dan memilih selector.

Selain itu, debugging jadi lebih mudah menggunakan TailwindCSS.

Memang, kombinasi TailwindCSS dan shadcn/ui sangat powerful untuk web development! ğŸ‘

## Kesimpulan

shadcn/ui bukan sekadar koleksi components. Ia adalah solusi praktis bagi developer yang menginginkan kebebasan, kontrol penuh, dan efisiensi dalam membangun UI components. Dengan pendekatan copy and paste, integrasi TailwindCSS, dan fokus pada accessibility, shadcn/ui telah membuktikan dirinya sebagai alat yang sangat berharga dalam pengembangan aplikasi modern.

Sebagai seorang software engineer yang sering berurusan dengan kompleksitas UI development, aku merasa shadcn/ui adalah salah satu tools yang bisa mengurangi frustrasi sekaligus meningkatkan produktivitas. Dengan kata lain, ia memberikan kita fondasi yang kuat untuk membangun UI yang tidak hanya cantik, tetapi juga inklusif dan scalable.
